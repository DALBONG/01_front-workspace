JavaScript 개요

1. script 언어란?

- 프로그램의 동작을 사용자의 요구에 맞게 수행되도록 해주는 용도


- (타 언어에 비해)매우 빠르게 배우고, 짧은 소스코드로 상호작용 되도록 고안

2. 자바 스크립트

웹 브라우저에서 많이 사용되는 "인터프리터 방식"의 스크립트 언어


- 인터프리터 방식 : 코드를 한 줄씩 읽어가며 바로 실행
(실시간으로 텍스트를 분석해서 실행)
=> 코드에 문법 오류가 있을 시 실행시점에서 알려줌

3. 장/단점

*장점

  • 컴파일의(통으로 읽음) 과정 없이, 한줄씩 읽어가며 바로 실행되어 수행속도가 빠름
  • 코드 작성이 간단하여 초보자가 접하기 쉬움(단순한 구조,원칙)

*단점

  • 웹에 특화된 기술로, 제공되는 기능 제한적임
  • html 소스코드 내에 같이 작성하게 되면 외부에 공개됨(보안 취약)

*4. 자바스크립트 사용법

  1. inline(인라인): 태그 내에 간단한 소스코드를 작성해서 실행
  2. internal(내부): html문서 내에 소스코드를 작성해서 실행
  3. external(외부): 별도의 파일 .js파일로 소스코드를 작성하고 가져다가 실행

1_인라인 방식

태그 내에 간단한 소스코드를 작성해서 실행
[표현법]< 태그 on 이벤트명="해당요소의 이벤트 발생시 실행할 소스코드">



2_인터널 방식

현재 html문서 내에 script 태그 영역 안에 소스코드 작성

- 스크립트 태그는 head / body 태그든 모두, 여러개 작성 가능

3_external 방식

별도의 .js파일을 만들어서 소스코드를 작성하여 가져다 쓰는 방식
